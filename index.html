General

What is Doge Yip?

Doge Yip is a decentrialized microbloging service that reads encoded data from the dogecoin blockchain using client-side javascript.

What do I need to start using Doge Yip?

All you need is a dogecoin wallet and some dogecoin. I would suggest going to dogecoin.com to learn more about dogecoin.

How do I post a yip to Doge Yip?

Click on the Yip link in the navagation bar, type whatever you want into the text field, and then send some dogecoin to each of the generated addresses. Once finished the yip is forever associated with the wallet that made the transactions.

How do I assign a name to my wallet?

Click on the Set Name link in the navagation bar, type whatever name you want into the text field, and then send some dogecoin to the generated address. Once finished that name is forever associated with the wallet that made the transaction.

Technical

What are the fees to send messages?

To send a message you must make a transaction to the generated address. To do so you would send over 1 Doge to the generated address and another Doge to a miner as incentive to put your transaction into the block chain. Note that the generated address has no known private key. Thus sending a Doge to that address is equivalent to burning a Doge.

What's the maximum message length?

There isn't a theoretical limit. Currently you are limited by how many transactions you want to commit for that message.

Some things to note: Currently chain.so only returns the last 50 transactions for a user. This means that if you use 10 transactions for one message then you are allocating 20% of your content to one post. Either chain.so will need to add in the ability to page transactions or I will need to launch my own block chain api to address this.

The end goal is for there to be no character limit. Currently the following is the longest message that I have successfully posted. It was encoded into 5 transactions:

two roads diverged in a yellow wood; and sorry I could not travel both; and be one traveler, long I stood; and looked down both as far as I could;

Won't this bloat the blockchain?

Potentially. But if a transaction amount is too small then no miner should actually write it into the block chain. I would think that it would produce no more bloat than the faucet sites.
